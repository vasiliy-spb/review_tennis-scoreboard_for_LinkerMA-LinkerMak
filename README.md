# Tennis Score Board

Веб-приложение для отслеживания счёта теннисного матча в реальном времени.

Проект реализует создание матчей, подсчёт очков по правилам тенниса, сохранение завершённых матчей и просмотр истории матчей с пагинацией. Приложение собрано на Spring MVC и Hibernate, в качестве базы данных используется H2. Для представлений используется Thymeleaf, для тестов — JUnit 5, деплой выполняется в Tomcat как WAR-артефакт.

## Функциональность

- Создание нового матча.
- Подсчёт очков по правилам тенниса:
  - очки внутри гейма;
  - счёт по геймам в сете;
  - счёт по сетам;
  - обработка deuce / advantage;
  - тай-брейк.
- Отображение текущего счёта матча в реальном времени.
- Сохранение завершённых матчей в базу данных.
- Просмотр списка завершённых матчей.
- Пагинация истории матчей.
- Поиск и просмотр статистики завершённых матчей.

## Архитектура

Приложение разделено на несколько логических слоёв:

- **Controller layer** — обработка HTTP-запросов и маршрутизация.
- **Service layer** — бизнес-логика матча и подсчёта счёта.
- **Persistence layer** — работа с БД через Hibernate.
- **View layer** — HTML-шаблоны на Thymeleaf.

Ключевая часть проекта — доменная логика теннисного счёта. Она инкапсулирует переходы между состояниями матча: розыгрыш очка, завершение гейма, завершение сета, запуск тай-брейка и завершение матча.

## Правила подсчёта счёта

В приложении поддержаны основные правила большого тенниса:

- стандартный порядок очков: 0, 15, 30, 40;
- ситуация **deuce**, когда оба игрока имеют по 40;
- состояние **advantage** после deuce;
- победа в гейме при преимуществе минимум в два очка;
- победа в сете при достаточном количестве выигранных геймов;
- тай-брейк при равном счёте по геймам, если он предусмотрен логикой матча.

## Запуск локально

### Требования

- JDK 21
- Maven 3.9+
- Apache Tomcat 11

### Сборка WAR

```bash
mvn clean package
```

После сборки WAR-артефакт будет находиться в директории `target/`.

### Запуск в Tomcat

1. Собрать проект:

```bash
mvn clean package
```

2. Скопировать WAR в директорию `webapps` Tomcat.

3. Запустить Tomcat.

После деплоя приложение будет доступно по адресу:

```text
http://localhost:8080/TennisScoreBoard
```

## Деплой на удалённый сервер

Проект можно задеплоить на удалённый Linux-сервер с установленными Java 21 и Apache Tomcat 11.

Общий сценарий деплоя:

1. Собрать WAR локально.
2. Подключиться к серверу по SSH.
3. Установить Java 21.
4. Установить Apache Tomcat 11.
5. Скопировать WAR в `/opt/tomcat/webapps/`.
6. Убедиться, что приложение развернулось.

Пример загрузки WAR на сервер:

```bash
scp -i <path-to-ssh-key> target/TennisScoreBoard.war <user>@<server-ip>:/opt/tomcat/webapps/
```

После деплоя приложение будет доступно по адресу:

```text
http://<server-ip>:8080/TennisScoreBoard
```
